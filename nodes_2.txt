### *PART 3: SERVER SETUP - NODE + EXPRESS BACKEND*

#### *STEP 1: Node Project Initialize Kiya*
mkdir server
cd server
npm init -y
*Theory:*
1. *`npm init -y`*: `package.json` file banata hai. `-y` matlab sab default values le lo
2. *`package.json` Kya Hai*: Project ka identity card. Naam, version, dependencies sab isme likha hota hai

#### *STEP 2: Backend Packages Install Kiye*
npm install express mongoose dotenv cors bcryptjs jsonwebtoken
*Theory - Har Package Ka Kaam:*
Package | Kaam | Interview Me Kya Bolna
**express** | Server banata hai. Routes like /api/jobs handle karta hai | "Express is a minimal Node.js web framework for building APIs"
**mongoose** | MongoDB se baat karne ka aasaan tarika | "Mongoose is an ODM that provides schema-based solution for MongoDB"
**dotenv** | .env file se secrets load karta hai | "It loads environment variables from .env file to process.env"
**cors** | Frontend-Backend ko connect karne deta hai | "CORS middleware allows cross-origin requests from React to Express"
**bcryptjs** | Password ko hash karta hai. Plain text me store nahi karte | "Bcrypt is used for password hashing for security"
**jsonwebtoken** | Login ke baad user ko Token deta hai | "JWT is used for stateless authentication"
#### *STEP 3: Server.js - Entry Point Banaya*
const express = require('express');
const app = express();
const PORT = 5000;

app.use(express.json()); // JSON data padh sake
app.use(cors());         // React ko allow kare

app.listen(PORT, () => console.log(`Server running on ${PORT}`));
*Theory:*
1. *`express()`*: Express app initialize kiya
2. *`app.use(express.json())`*: Middleware. POST request me jo JSON data aata hai usko `req.body` me convert karta hai
3. *`cors()`*: Agar ye nahi lagaya to React `localhost:5173` se Express `localhost:5000` pe request nahi bhej payega. Browser block kar dega
4. *`app.listen`*: Server start karta hai PORT 5000 pe

#### *STEP 4: .env File - Secrets Store Kiye*
MONGO_URI=mongodb+srv://username:password@cluster.mongodb.net/indorejabs
JWT_SECRET=mysecretkey123
PORT=5000
*Theory: .env Kyu Chahiye?*
1. *Security*: Password code me hardcode nahi karte. GitHub pe chala gaya to hack ho jayega
2. *Environment*: Development me alag DB, Production me alag DB use kar sakte hain
3. *Access*: Code me `process.env.MONGO_URI` likh ke access karte hain

*Interview Q*: ".env file git me kyu nahi daalte?" 
*Ans*: "Because it contains sensitive credentials like DB passwords and API keys. If exposed publicly, it can lead to security breach. That's why we add it to .gitignore."

#### *STEP 5: MongoDB Connect Kiya*
const mongoose = require('mongoose');
mongoose.connect(process.env.MONGO_URI)
.then(() => console.log("MongoDB Connected"))
.catch(err => console.log(err));
*Theory:*
1. *Mongoose*: MongoDB ka driver. MongoDB se connect karne aur query karne me help karta hai
2. *Connection String*: `mongodb+srv://...` ye MongoDB Atlas ka URL hai. Cloud database ka address
3. *Async*: Database connect hone me time lagta hai, isliye `.then()` use karte hain

### *PART 5: INTERVIEW QUESTIONS - SETUP WALE*

*Q1: Vite vs Create-React-App?* 
*Ans*: Vite is faster because it uses ES modules and doesn't bundle everything in development. CRA uses Webpack which is slower. For production both are similar.

*Q2: package.json vs package-lock.json?* 
*Ans*: `package.json` lists dependencies with version range like ^18.2.0. `package-lock.json` locks exact version like 18.2.3. This ensures everyone in team installs same versions.

*Q3: Express me middleware kya hota hai?* 
*Ans*: Middleware are functions that run between request and response. Example: `express.json()` parses incoming JSON, `cors()` handles cross-origin. They have access to req, res, next.

*Q4: MongoDB SQL se kaise alag hai?* 
*Ans*: MongoDB is NoSQL, document-based. Data JSON-like objects me store hota hai. SQL me tables-rows hote hain. MongoDB is flexible schema, good for rapid development.

*Q5: JWT kaise kaam karta hai?* 
*Ans*: After login, server creates a JWT token with user ID and signs it with secret. Client stores it and sends in Authorization header for future requests. Server verifies token without checking DB every time. This is stateless auth.

*Q6: node_modules git me kyu nahi daalte?* 
*Ans*: 1. Very heavy, 50k+ files. 2. Can be regenerated with `npm install` using package.json. 3. OS specific files ho sakte hain. Pushing it wastes space and time.
