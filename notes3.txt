### *PART 1: BASIC THEORY - INTERVIEW ME PUCHNE WALE CONCEPTS*

#### *1. Git Kya Hai?*
- *Definition*: Git ek *Version Control System (VCS)* hai. Matlab code ka history maintain karta hai.
- *Real Life Example*: Jaise MS Word me `Ctrl+Z` karke undo karte ho, waise hi Git me purane code pe wapas ja sakte ho.
- *Kyu Chahiye*: Agar code bigad jaye, client ko purana version chahiye, ya team me kaam karna ho to Git zaroori hai.

#### *2. GitHub Kya Hai?*
- *Definition*: GitHub ek *Cloud Platform* hai jahan Git repositories store karte hain.
- *Git vs GitHub*:
Git | GitHub
Software, laptop me chalta hai | Website, internet pe chalta hai
Local version control | Remote backup + collaboration
Command line tool | UI + Git features
#### *3. Repository (Repo) Kya Hai?*
- *Definition*: Project ka folder jise Git track kar raha hai.
- *2 Types*: 
    1. *Local Repo*: Tere laptop me `indorejabs` folder
    2. *Remote Repo*: GitHub pe `shraddha-stack-debug/indore-jobs-portal`

#### *4. Commit Kya Hai?*
- *Definition*: Code ka ek *snapshot/save point*. Har commit ka unique ID hota hai.
- *Example*: `git commit -m "Login page banaya"`
- *Interview Line*: "Commit ek checkpoint hai. Agar naya code galat ho jaye to purane commit pe wapas ja sakte hain."

#### *5. Branch Kya Hai?*
- *Definition*: Code ki *alag line*. `main` branch production code hota hai.
- *Kyu Chahiye*: Naya feature banate time `main` ko mat chhedo. Alag branch me kaam karo.
- *Default Name*: Pehle `master` tha, ab `main` use hota hai.

#### *6. .gitignore Kya Hai?*
- *Definition*: Wo file jisme likhte hain ki Git ko kya ignore karna hai.
- *Tere Project Me Kyu Daala*:
    1. `node_modules/` = 50,000 files, 200MB+. GitHub pe upload karna waste hai
    2. `.env` = Isme MongoDB ka password hai. Public ho gaya to hack ho jayega
    3. `build/` = Ye auto-generate hota hai. Code se ban jata hai
- *Interview Line*: "Sensitive files aur auto-generated files ko .gitignore me daalte hain."

### *PART 2: TUNE KYA-KYA KIYA - STEP BY STEP WITH THEORY*

#### *STEP 1: `git init` - Repository Initialize Kiya*
git init
- *Theory*: Is command se `indorejabs` folder me ek hidden `.git` folder ban jata hai. Yahi folder poora history store karta hai.
- *Kab Use Kare*: Jab naya project start kare aur version control chahiye ho.
- *Output*: `Initialized empty Git repository in C:/Users/.../indorejabs/.git/`

#### *STEP 2: `.gitignore` Banaya*
- *Theory*: Git ko bataya ki `node_modules` aur `.env` ko track mat karna.
- *Important Rule*: `.gitignore` file sabse pehle banao, `git add .` se pehle. Warna file track ho jayegi.

#### *STEP 3: `git add .` - Staging Area Me Daala*
git add .
- *Theory*: Git me 3 areas hote hain:
    1. *Working Directory*: Jahan tu code likh rahi hai
    2. *Staging Area*: `git add` ke baad files yahan aati hain. Matlab "ye files commit ke liye ready hain"
    3. *Repository*: `git commit` ke baad files yahan permanent save ho jati hain
- *`.` ka Matlab*: Current folder ki sabhi files add karo
- *Interview Q*: "Staging area kyu hota hai?" → *Ans*: Taaki hum select kar sakein ki konsi files commit karni hain. Sab ek saath commit karna zaroori nahi.

#### *STEP 4: `git commit -m "message"` - Snapshot Save Kiya*
git commit -m "Initial commit: Indore Jobs Portal setup"
- *Theory*: Staging area se files uthakar Local Repository me permanent save kar diya.
- *Message Kyu Zaroori*: Baad me dekhne ke liye ki is commit me kya change kiya tha.
- *Good Message Example*: `"Add user login API"` `"Fix navbar bug"`
- *Bad Message*: `"update"` `"changes"`
- *Output Tera*: `25 files changed` = Matlab 25 files ka pehla version save ho gaya

#### *STEP 5: GitHub Pe Repo Banaya*
- *Theory*: Remote repository banayi taaki code ka backup cloud pe rahe aur duniya ko dikha sake.
- *Public vs Private*: 
    - *Public*: Sab dekh sakte hain. Portfolio ke liye best
    - *Private*: Sirf tujhe dikhega. Company ke project ke liye

#### *STEP 6: `git remote add origin URL` - Link Kiya*
git remote add origin https://github.com/shraddha-stack-debug/indore-jobs-portal.git
- *Theory*: Tere Local Repo ko bataya ki Remote Repo ka address ye hai.
- *`origin` Kya Hai*: Remote ka nickname. Default name `origin` hota hai.
- *Interview Q*: "Ek se zyada remote ho sakte hain?" → *Ans*: Haan. `origin`, `upstream` etc.

#### *STEP 7: `git branch -M main` - Branch Ka Naam Set Kiya*
git branch -M main
- *Theory*: `-M` matlab force rename. Branch ka naam `main` kar diya.
- *Kyu Zaroori*: GitHub ka default branch `main` hota hai. Agar tera local `master` hai to conflict hoga.

#### *STEP 8: `git push -u origin main` - GitHub Pe Upload Kiya*
git push -u origin main
- *Theory*: Local `main` branch ka code Remote `origin` ke `main` branch pe bhej diya.
- *`-u` Kya Hai*: `--set-upstream` ka short form. Matlab future me bas `git push` likhne se kaam ho jayega. Baar-baar `origin main` nahi likhna padega.
- *Git Credential Manager*: Ye popup isliye aaya kyunki GitHub ko proof chahiye ki tu hi hai. Browser me login karke tu authorize karti hai.

### *PART 3: WORKFLOW - DAILY LIFE ME KAISE USE HOGA*

*Maan le kal tune login page me change kiya. To ye karegi:*

1. *Code Change Karo* `client/login.js` me
2. *Check Karo Kya Change Hua*
   git status
Output: `modified: client/login.js`

3. *Stage Karo*
   git add client/login.js
Ya sab ek saath: `git add .`

4. *Commit Karo*
   git commit -m "Add password show/hide feature in login"
5. *Push Karo*
   git push
Bas. GitHub pe update ho gaya.

### *PART 4: INTERVIEW QUESTIONS + ANSWERS*

*Q1: Git aur GitHub me difference?* 
*Ans*: Git ek version control tool hai jo local me chalta hai. GitHub ek cloud service hai jahan hum Git repositories host karte hain for backup and collaboration.

*Q2: `git add` aur `git commit` me difference?* 
*Ans*: `git add` files ko staging area me le jata hai matlab commit ke liye ready karta hai. `git commit` staging area se files ko local repository me permanent save kar deta hai ek snapshot ke saath.

*Q3: .gitignore kyu use karte hain?* 
*Ans*: Sensitive data jaise .env file, heavy folders jaise node_modules, aur auto-generated files jaise build ko track hone se bachane ke liye. Ye security aur performance dono ke liye important hai.

*Q4: `git push -u origin main` me -u ka kya matlab hai?* 
*Ans*: `-u` ya `--set-upstream` local branch `main` ko remote `origin/main` se link kar deta hai. Iske baad future me sirf `git push` likhne se kaam ho jata hai.

*Q5: Agar galti se .env file push ho jaye to kya karenge?* 
*Ans*: 1. Turant GitHub se delete karenge. 2. .gitignore me add karenge. 3. `git rm --cached .env` karke git se remove karenge. 4. MongoDB ka password change karenge kyunki wo leak ho gaya.

*Q6: Commit message kaise hone chahiye?* 
*Ans*: Clear aur descriptive. Jaise "Fix: Navbar overlap issue on mobile" ya "Feat: Add JWT authentication". "update" ya "changes" jaise vague message nahi.
